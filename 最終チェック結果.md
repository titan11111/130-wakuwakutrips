# 出張プログラム視認化ツール - 最終デバッグチェック結果

## 1. 携帯での視認性（文字・カレンダー）

### 対応内容
- **ドロップ後の地域名（「熊本」等）**: 携帯で小さくなりすぎないようフォントサイズの下限を設定
  - ベース: `.cell-location-item` を 0.85rem、`.cell-location` を 0.95rem に変更
  - 768px以下: 地域名を **0.75rem 以上**に統一（0.6rem → 0.75rem）
  - 480px以下: **0.75rem** を維持
  - 縦短い画面（max-height: 700px）: **0.7rem 以上**に変更（0.5rem → 0.7rem）
- セル内パディング・gap を少し広げ、2つ並んでも読みやすく調整

### 確認ポイント（実機）
- カレンダーに「熊本」「東京」等をドロップした後、スマホで文字がはっきり読めるか
- 1マスに2つ並んだときも両方読めるか

---

## 2. 音楽（BGM）

### 現状・対応
- **再生開始**: 「出張する？」ボタン（ユーザージェスチャー）で `Sound.init()` と `BGM.start()` を実行 → iOS の自動再生制限に対応
- **再生失敗時**: 初回 `play()` が失敗した場合に約 150ms 後に再試行するように変更（一部 iOS で再生が遅れる場合の対策）
- **タブ復帰**: `visibilitychange` / `pageshow` で BGM と AudioContext を再開済み

### 確認ポイント（実機）
- スタート画面で「出張する？」をタップ後、BGM が流れ始めるか
- 別タブに移って戻ったとき、BGM が再開するか（ミュート・作業モードでない場合）

---

## 3. PDF 化のスムーズ化

### 対応内容
- **レイアウト待ち**: `html2canvas` 実行前に `requestAnimationFrame` を2フレーム待機し、オフスクリーンコンテナのレンダリングを安定させた
- **オプション**: `logging: false` でコンソール出力を抑制、`windowWidth` / `windowHeight` でキャンバスサイズを明示
- **ローディング表示**: オーバーレイの `z-index` を 10000 にし、PDF 生成中は常に最前面に表示

### 確認ポイント（実機）
- 「PDFで決定」押下後、「PDFを生成中...」が確実に表示されるか
- 生成が完了するまで待機し、PDF が問題なくダウンロードされるか

---

## 4. ドラッグ＆ドロップ（携帯・タッチ）

### 対応内容
- **タッチ用フォールバック**: iOS など HTML5 DnD が不安定な環境向けに、タッチ専用のドラッグ＆ドロップを実装
  - **地域タグ → カレンダー**: タグをタッチして動かすとゴーストが指に追従し、離した位置のマスにドロップで配置
  - **カレンダー → 地域エリア**: 配置済みセルをタッチして動かし、右側の「出張地域を選ぶ」エリアに離すと配置解除
- ゴーストは `.touch-drag-ghost` でスタイル統一（フォント・タップハイライト無効）
- マウス操作は従来どおり HTML5 DnD で動作（両方有効）

### 確認ポイント（実機）
- 地域タグを長押しして動かし、カレンダーのマスで離すと配置されるか
- カレンダーに配置した「熊本」等のセルを長押しして、右の地域エリアで離すと解除されるか
- 1マスに2つまで配置できるか、ドロップ時トーストで「2つまで」と出るか

## チェック一覧（実機推奨）

| 項目 | 内容 |
|------|------|
| 視認性 | 携帯でカレンダー・地域名が読みやすいか |
| BGM | 「出張する？」で再生開始、タブ復帰で再開するか |
| PDF | 生成中表示が出て、PDF がダウンロードできるか |
| DnD（タッチ） | タグ→セル・セル→地域エリアのドラッグができるか |
| 効果音 | ボタン・ドロップ時にぴこ音が鳴るか（作業モード時は鳴らないか） |
| セーフエリア | ノッチ・ホームインジケータでボタンが隠れていないか |

以上を実機（iPhone / Android）で確認すると安心です。
